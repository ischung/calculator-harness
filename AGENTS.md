# AGENTS.md — AI 에이전트 운영 지침

> **이 파일은 AI 에이전트가 이 저장소에서 작업할 때 따라야 할 모든 규칙을 담는다.**
> 약 100줄 이하로 유지한다. 깊은 내용은 각 링크를 따라가라.

## 에이전트 역할 정의

```
[ROLE-1] 하네스 에이전트: 구조적 제약 강제 및 엔트로피 관리
[ROLE-2] 계산 에이전트: core/ 함수 구현 및 수학적 정확성 검증
[ROLE-3] 검증 에이전트: validation/ 레이어 유지 및 보안 경계 강화
[ROLE-4] 문서 에이전트: docs/ 일관성 유지 및 stale 문서 감지
```

## 코딩 규칙

### 필수 규칙
1. **나눗셈 함수**는 반드시 분모가 0인지 먼저 검사한다.
2. **계산 함수**는 순수 함수(pure function)여야 한다. 부수 효과(side effect) 금지.
3. **타입**은 반드시 `types/` 에서 import한다. 인라인 타입 정의 금지.
4. **에러 메시지**는 사용자가 이해할 수 있는 한국어와 영어를 병기한다.

### 금지 사항
- `ui/` 에서 `core/` 직접 import (반드시 서비스 레이어 경유)
- `try-catch`로 0 나누기 예외 무시
- 검증 없는 사용자 입력 직접 사용
- 주석 없는 수학 공식 구현

## 피드백 루프 (Feedback Loop)

에이전트는 모든 작업 완료 후 반드시 다음을 자체 검증한다:

```bash
# 1단계: 불변 규칙 검증
npm test -- --grep "invariants"

# 2단계: 의존성 방향 린팅
npm run lint:deps

# 3단계: 타입 안전성
npm run typecheck

# 4단계: 수학적 정확성 eval
npm test -- --grep "evals"
```

CI가 실패하면 에이전트는 스스로 오류를 읽고 수정한다. 인간에게 에스컬레이션하기 전에 **최소 2회 자체 수정**을 시도한다.

---

## 엔트로피 관리 규칙

- **백그라운드 에이전트**는 주기적으로 다음을 스캔한다:
  - `docs/` 문서와 코드의 불일치
  - 중복 패턴 (DRY 위반)
  - `types/`에 없는 인라인 타입 정의
  - 미사용 함수 및 export

- 발견 시: `docs/exec-plans/active/` 에 리팩토링 계획 자동 생성
